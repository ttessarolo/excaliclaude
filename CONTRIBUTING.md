# Contribuire a excaliclaude

> 🇬🇧 **English:** Anyone acting in good faith is welcome to test and improve this project. Contributions, bug reports, and ideas are all appreciated. Read below (Italian) or just open an issue — we'll figure it out together.

Grazie per l'interesse verso **excaliclaude**! Questo è un progetto aperto: **chiunque di buona volontà è benvenuto** a testarlo, segnalare problemi, proporre miglioramenti o contribuire con codice. Non servono credenziali particolari — basta voglia di collaborare e rispetto reciproco.

## Come puoi contribuire

Ci sono molti modi utili, anche senza scrivere codice:

- **🐛 Segnalare bug** — apri una [Issue](../../issues/new) descrivendo cosa ti aspettavi, cosa è successo, e i passi per riprodurre il problema.
- **💡 Proporre funzionalità** — apri una Issue con label `enhancement` spiegando il caso d'uso.
- **📖 Migliorare la documentazione** — correzioni, chiarimenti, traduzioni: tutto benvenuto.
- **🧪 Testare** — prova il progetto in scenari reali e condividi feedback.
- **💻 Inviare codice** — fix, feature, refactoring via Pull Request.

## Setup locale

Per installare e avviare il progetto in locale, segui le istruzioni nel [README](./README.md).

## Inviare una Pull Request

1. **Fai un fork** del repository e crea un branch descrittivo:
   ```bash
   git checkout -b fix/descrizione-breve
   # oppure
   git checkout -b feat/nuova-funzionalita
   ```
2. **Scrivi i tuoi cambiamenti** mantenendo lo stile del codice esistente.
3. **Testa manualmente** le modifiche prima di aprire la PR.
4. **Commit chiari** — consigliato il formato [Conventional Commits](https://www.conventionalcommits.org/):
   - `feat: aggiungi supporto per X`
   - `fix: risolvi crash quando Y`
   - `docs: aggiorna README per Z`
5. **Apri la Pull Request** includendo:
   - **Cosa** cambia e **perché**
   - **Come testarla** (passi per verificare il comportamento)
   - Eventuali screenshot/GIF per modifiche UI
6. **Mantieni la PR piccola e focalizzata** — una PR = un obiettivo. Più facile da revisionare, più veloce a essere unita.

## Testing

Prima di aprire una PR:

- Verifica che il progetto compili e parta senza errori.
- Prova manualmente il flusso che hai modificato, incluso almeno un caso limite.
- Se hai rotto qualcosa di non correlato, segnalalo nella descrizione.

## Code of Conduct

Ci aspettiamo che tutti i partecipanti si comportino con rispetto e professionalità:

- **Sii gentile** e costruttivo nelle discussioni.
- **Niente tolleranza** per molestie, insulti, discriminazioni o comportamenti tossici.
- **Assumi buona fede** negli altri collaboratori.
- **Critica il codice, non le persone.**

Comportamenti inappropriati possono comportare la rimozione dei contributi o il ban dal progetto.

## Domande?

Per qualsiasi dubbio, apri una [Issue](../../issues/new) con label `question`. Nessuna domanda è stupida — se qualcosa non è chiaro, probabilmente non lo è per molte persone.
